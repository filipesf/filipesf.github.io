Checkout form

The design process behind the Wordpress plugin, WP Pusher, checkout page redesign.

Use case

I have been developing websites with Wordpress for quite some time, and in one of many newsletters that I subscribe, I found WP Pusher plugin. On the newsletter, there was just a quick headline, a six words description and a link. All that in an email full of other good stuff. From there I was sent straight to the checkout page with a basic plan selected and a couple of input fields to fill out, but all I knew so far was that WP Pusher was a good plugin.

Product

WP Pusher lets you install and update WordPress themes and plugins, directly from GitHub, Bitbucket and GitLab - even if your clients are using cheap shared hosting. If it can run WordPress, it can run WP Pusher. You can enable Push-to-Deploy to have your code update automatically every time you push to Git.

Pain points

My experience as a customer didn’t start very well, but in fairness, that is not all WP Pusher’s fault. Altough the plugin’s checkout page could mitigate a good chunk of misunderstandings, the way the advertisement was made was not ideal. I listed 11 pain points that could increase significantly the conversion on this checkout page.

1. Lack of context

The checkout form is floating in a blank page and the user has not enough information about the context he/she is in or even more details about the product itself.

2. Reading direction

Although the WP Pusher logo is helping the user identify from where he/she is buying the product from, the centered alignment does not help the user scan the page, misleading he/she to read “Checkout” before even identifying the brand.

3. Too little details

The dropdown menu only shows minimal details about the plan the user is choosing. Also, it doesn’t show the user all the options he/she has at glance, which does not help the user choose the plan that better fit his/her needs.

4. Total sum

The “Total sum” section is a repetition of the price that is displayed on the dropdown menu. Doesn’t matter which plan you choose, it will always be the same in both places.

5. Hidden information

The renewal section is really important but is grayed out, not giving it the real importance. From the users perspective, is crucial for him/her to konw that the subscription plan will be renewed automatically after 1 year as it might be decisive whether or not the user will subscribe to that plan.

6. The ruler

The ruler is meant to separete the plan details form the user information sections. However, there is no indication what the next section is about. Although the ruler does the job of separate different sections, it does not indicate the what is coming next.

7. Personal details

In this section we see the placeholders are being used as labels and this brings many usability issues. Disappearing placeholder text strains users’ short-term memory. Without labels, users cannot check their work before submitting the form. When error messages occur, people don’t know how to fix the problem.

8. Payment section

Once again we have the ruler doing its job, however, it has a title that indicates the user what he/she has coming up on the following section. The issue here is mainly the inconsistency when the user interacts with two different sections.

9. Payment information

The placeholders is still being used as labels and on top of that, this is the most sensitve step where the user put all his/her trust on the company by sharing their credit card information. With that in mind, the required information must be clear. On the “Expires” field, the placeholder text indicates “mm/yy”, but when the user tries to fill the date, it actually requires “mm/yyyy”. The acronym “CVC” can vary from a card brand to another as each one has their own naming convention and it can confuse the user when he/she tries to find this information on their credit card.

10. Call-To-Action

The call-to-action copy may indicate that, once the user pays, he/she will have the WP Pusher license, however, the user is actally subscribing to their tool, thus not acquiring a life-time license.

11. Contact support

The “need any help” section is being displayed in a coloured box, right below the call-to-action button and it may indicate that the user will only get help after he/she finishes the purchase. It also takes out the attention from the call-to-action button, leading to a potential doubt the user may have, which can make the conversion longer than expected or even not converting at all.
